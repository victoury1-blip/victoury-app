# دليل تكامل WooCommerce

## نظرة عامة
تم ربط نظام إدارة المخزون الخاص بك مع WooCommerce. عندما تضيف أو تعدل أو تحذف منتج معين في نظام المخزون، سيتم مزامنته تلقائياً مع متجر WooCommerce الخاص بك.

## الميزات
- **إضافة منتجات**: عند إضافة منتج مع اختيار "WooCommerce" كمتجر، يتم إنشاؤه تلقائياً في WooCommerce مع جميع الأحجام
- **تحديث المنتجات**: أي تغيير على اسم المنتج أو السعر أو الحالة يتم مزامنته تلقائياً
- **حذف المنتجات**: حذف منتج من نظام المخزون يحذفه من WooCommerce أيضاً
- **مزامنة المخزون**: تعديل الكمية يتم مزامنته تلقائياً مع WooCommerce
- **سجل المزامنة**: جميع العمليات يتم تسجيلها للمراجعة

## خطوات الإعداد

### 1. الحصول على بيانات اعتماد WooCommerce API

1. اذهب إلى لوحة تحكم WooCommerce
2. انتقل إلى **Settings > Advanced > REST API**
3. انقر على **Create an API key**
4. أدخل التفاصيل:
   - **Description**: Victoury Stock System
   - **User**: اختر مستخدم المسؤول
   - **Permissions**: Read/Write
5. انقر على **Generate API key**
6. انسخ **Consumer Key** و **Consumer Secret**

### 2. تكوين متغيرات البيئة

1. انسخ `.env.example` إلى `.env.local`:
   ```bash
   cp .env.example .env.local
   ```

2. عدّل `.env.local` وأضف بيانات WooCommerce:
   ```
   VITE_WOO_API_URL=https://your-woocommerce-site.com
   VITE_WOO_CONSUMER_KEY=your_consumer_key_here
   VITE_WOO_CONSUMER_SECRET=your_consumer_secret_here
   ```

3. أعد تشغيل خادم التطوير

## كيفية الاستخدام

### إضافة منتج
1. انقر على زر **Ajouter** (إضافة)
2. أدخل تفاصيل المنتج
3. اختر **WooCommerce** من قائمة "Boutique"
4. أضف الأحجام والكميات
5. انقر على **Ajouter le produit**
6. سيتم المزامنة تلقائياً مع ظهور رسالة نجاح

### تعديل منتج
1. انقر على زر التعديل (أيقونة القلم)
2. اجعل التغييرات المطلوبة
3. انقر على **Enregistrer** (حفظ)
4. سيتم المزامنة تلقائياً

### حذف منتج
1. انقر على زر الحذف (أيقونة سلة المهملات)
2. أكد الحذف
3. سيتم حذف المنتج من كلا النظامين

### تعديل المخزون
1. انقر على **Voir variations** (عرض الأحجام)
2. استخدم أزرار +/- لتعديل الكمية
3. انقر على العلامة الخضراء للتطبيق
4. سيتم المزامنة تلقائياً مع WooCommerce

## مؤشرات حالة المزامنة

يظهر في أعلى الصفحة:

- **أزرق مع دوران**: جاري المزامنة
- **أخضر مع علامة**: نجحت المزامنة
- **أحمر مع X**: فشلت المزامنة

## استكشاف الأخطاء

### "WooCommerce not configured"
- تأكد من وجود `.env.local` وأنه يحتوي على البيانات الصحيحة
- تحقق من صحة بيانات اعتماد API
- أعد تشغيل خادم التطوير

### "Failed to create product"
- تأكد من أن مفتاح API لديك له صلاحيات Read/Write
- تحقق من تفعيل REST API في WooCommerce
- تحقق من صحة رابط API

### المخزون لا يتم مزامنته
- تأكد من أن المنتج معلم كـ "WooCommerce"
- افتح console في DevTools (F12) وابحث عن الأخطاء
- تحقق من صحة بيانات اعتماد API

## ملاحظات أمان

- **لا تحفظ `.env.local` في Git** - يحتوي على بيانات حساسة
- بيانات الاعتماد مخزنة في متغيرات البيئة فقط
- لا تُرسل بيانات الاعتماد إلا عبر HTTPS
- استخدم مفتاح API منفصل بصلاحيات محدودة

## الدعم

إذا واجهت مشاكل:
1. اقرأ رسالة حالة المزامنة
2. افتح console في DevTools (F12) وابحث عن الأخطاء
3. تحقق من بيانات اعتماد WooCommerce
4. تأكد من أن موقع WooCommerce متاح
5. تحقق من تفعيل REST API في إعدادات WooCommerce
